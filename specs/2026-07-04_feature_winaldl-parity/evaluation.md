### Persona Reviews

**Product Manager** — Scope matches the agreed Phase 3 cut exactly: steps 3.1–3.4, the three binding user decisions (prompt on all three actions; WinALDL spark axes; Spark as tab 5), and the post-implementation notice-expiry feedback are all implemented; the deferred "editable save filename" memory item is fulfilled (prompt pre-fills `goaldl_<ts>`, Enter accepts, Esc cancels). No creep: `monitor` gains no spark view or runtime toggles, no `serve` adapter, no dash view, no config persistence — all correctly left per spec §9. User value at the car is real: the operator can now start a raw capture *after* seeing something interesting (previously required restarting with `record`), scrub a replay while studying a grid, and localize knock on a finer MAP axis. One rough edge for the user: the pre-parse waiting message still points to "7 Raw" when Raw is now tab 8 (Issue 1) — briefly misleading exactly when a new user is most likely to see it.

**Architect** — The layering discipline is the strongest part of this change. Both runtime controls land *below* the Session facade on the provider layer (`RecordSink` via the pre-existing `Sink` field — the SerialProvider itself needed zero changes; pause/speed as mutex-guarded `ReplayProvider` methods with the exported `Speed` field's meaning preserved), so a future `serve` front-end inherits both for free and `Snapshot` stays plain data. The re-anchoring pacing algorithm in `Run` is correct on inspection: progress is captured into `anchorData` only on control transitions while running, the paused branch pins the data position by re-pinning `anchorWall`, and bounded 100ms wait slices give responsive control pickup without busy-waiting; the injectable-clock tests prove the arithmetic to ±1ms against the real fixture. `gridHeat` generalizing to a values matrix (rather than a parallel spark renderer) and `saveGrids` moving to a table-driven four-file loop are consolidation done right. No new dependencies; no API contract breakage (`BLMBody`/`SensorTable` signatures intact, `gridHeat` is unexported). Nits: `LoopStatus` passing `dot(o2On)` twice is intentional (spark shares O2's condition) but a reader could take it for a copy-paste bug — a named `sparkOn := hasGood` would self-document; `closeOutputs`'s detach-before-close ordering is the correct concurrency answer for quit-while-recording.

**QA** — Coverage against the spec's §8 test plan is essentially complete: every row maps to a named passing test, including the ones that usually get skipped (concurrent Write/Set under `-race`, the 250→4 wrap, sum-vs-samples distinction, collision keeping the prompt open, stale-timer notice sequencing, clamps at both 0.25× and 16×). The knock baseline is protected from bad frames by the ParseOK early-return in `accumulate` — the one path that would have silently corrupted spark counts — and clear-keeps-baseline is asserted. Remaining soft spots, none blocking: `closeOutputs` (quit with recording/CSV open) has no direct test — the detach-then-close ordering is right by inspection but unguarded against regression; the sink write-error notice only surfaces when a next snapshot arrives (a stalled port leaves the REC badge up until then); CSV collision is stat-then-`os.Create` with a truncating race window (spec-sanctioned to avoid touching shared csv.go, but it is the one non-`O_EXCL` path); `TestTUIPromptEditing`'s esc-cancel assertion only inspects the temp dir, so a wrong write into the CWD would escape it; and a mid-write failure in `saveGrids` (disk full on file 3 of 4) leaves a partial set behind with only a notice. The knock-counter-reset spurious delta is a documented, accepted spec edge (§7), noted in the `SparkBody` doc comment as required.

### Issues Found

1. **Stale tab reference in the waiting message** — `cmd/goaldl/tui.go:742`: `"waiting for a parseable frame… (see 7 Raw for the byte stream)"` — Raw is tab **8** after the renumber; every other reference (tab bar, key map, help footer `1-8/tab`) was updated. Severity: **warning**.
2. **CLAUDE.md not updated** — `/Users/aaronstone/Development/aldl/goaldl/CLAUDE.md:37,105` still describes "seven tabs (keys 1-7)", "`s` saves all three grids to timestamped files" (now four, via prompt), and lists Phase 3 as "next". The brief scopes docs out of code evaluation (and README/tasks/PROJECT_STATUS *were* updated), but CLAUDE.md steers every future session and now contradicts the shipped behavior. Severity: **warning** (fix before/with commit).
3. **CSV collision check is stat-then-create (TOCTOU)** — `confirmPrompt` promptCSV path (`cmd/goaldl/tui.go`, `os.Stat` then `newFrameCSV` → `os.Create`, which truncates). A file created externally in the window is silently overwritten. Spec §2 explicitly sanctions "stat first" to keep csv.go shared with `monitor -csv` untouched, so this is contract-compliant. Severity: **note**.
4. **Sink write-error surfacing is snapshot-driven** — the `recSink.Err()` check lives in the snapshotMsg handler (`cmd/goaldl/tui.go` ~line 379); if the port stalls after the error, the stop notice and handle close wait for the next frame (or quit, where `closeOutputs` cleans up). On a live source frames arrive ~1.2s apart, so exposure is minimal. Severity: **note**.
5. **`saveGrids` can leave a partial file set on a non-collision error** — `cmd/goaldl/tui.go` `saveGrids`: the pre-flight stat pass guards collisions, but a create/write failure on file N leaves files 1..N−1 on disk with only a `save failed` notice. Spec only promises clean abort on *collision*. Severity: **note**.
6. **No direct test for `closeOutputs`** (quit with active recording/CSV) — `cmd/goaldl/tui.go`; the Set(nil)-before-Close ordering is correct but regression-unguarded, and it's the one path the Bubble Tea model tests can't reach without running the program. Severity: **note**.
7. **`TestTUIPromptEditing` esc-cancel assertion scope** — `cmd/goaldl/tui_test.go`: checks only the temp dir is empty; a buggy write to the working directory (the default base has no dir component) would pass. Severity: **note** (test quality).
