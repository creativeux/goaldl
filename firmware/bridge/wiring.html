ALDL → ESP32 input conditioning

One transistor stands between the car's 12V-domain data line and the QT Py's 3.3V-only RX pin. It clamps the voltage and inverts the signal — the same two jobs the PL2303 cable's internal circuit does today, so the bytes come out identical to a serial capture.

Measure before wiring. Key on, engine off: pin E to pin A should read a few volts and be busy (the ECM streams continuously). If it reads 0V or battery voltage steadily, re-check which pin you're on before connecting anything.

  • Power in the car: 12V accessory → any USB adapter → the QT Py's USB-C. USB carries power only in the deployed setup; data leaves over WiFi.
  • Optocoupler alternative (PC817 + ~1kΩ LED resistor) buys full galvanic isolation for a couple more parts — same inverting behavior, same three board pins.
  • The bridge firmware needs no change for any of this: it reads RX at 4800 baud and forwards bytes verbatim, warts and all.

One connector, both ALDL generations superset design

The 8192-baud ECMs (mid-'86+) use the same 12-pin shell but move data to pin M as a half-duplex request/response UART. One built cable can serve both: wire pin E and pin M, each through its own stage, to two of the S3's three UARTs — firmware selects which is live.
